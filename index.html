Explain javascript array methods: push, pop, shift and unshift. Show how to use javascript array as queue and stack respectively.

push: adds one or more items to the end of an array.

pop: removes and returns the last item from an array.

shift: removes and returns the first item from an array.

unshift: adds one or more items to the beginning of an array.

To use an array as a queue, add items to the end using push, and remove them from the beginning using shift.

To use an array as a stack, add items to the end using push, and remove them from the end using pop.

What are data-attributes?

Data attributes are custom attributes in HTML that start with "data-" and can store extra information or metadata about the element. They are useful for storing configuration data for JavaScript code, providing metadata for search engines, and adding custom styling or functionality. In JavaScript, data attributes can be accessed and modified using the "dataset" property of the element.

Asynchronous, await, vs. synchronous, blocking calls vs. noblocking calls

Synchronous calls block program execution until an operation is complete, while asynchronous calls and non-blocking calls allow the program to continue executing other instructions while the operation is being performed. The "await" keyword simplifies working with asynchronous calls.

React components, state, props, jsx, rxjs, virtual DOM

Components - React is built around the concept of reusable components, which are small, self-contained units of code that can be composed together to build complex user interfaces.

State - A component's state is an object that holds information about the component's current state, which can change over time in response to user actions or other events.

Props - Props (short for "properties") are passed down from a parent component to a child component, and are used to provide data and configuration to the child component.

JSX - JSX is a syntax extension that allows developers to write HTML-like code in their JavaScript code, making it easier to write and reason about UI components.

RxJS - RxJS is a library for reactive programming that is often used in conjunction with React. It provides a way to handle asynchronous events and data streams in a more declarative and composable way.

Virtual DOM - React uses a virtual DOM, which is an in-memory representation of the actual DOM. The virtual DOM allows React to perform updates to the UI in a more efficient way by minimizing the number of DOM operations needed.

session and sessionStorage, localStorage, IndexedDB

Session and sessionStorage - Session and sessionStorage are both mechanisms for storing data on the client-side that is specific to a particular session or tab in the browser. Session data is stored in the browser's memory and is cleared when the session is ended, while sessionStorage data is stored in the browser's memory but is cleared when the tab is closed.

LocalStorage - LocalStorage is a mechanism for storing data on the client-side that persists even after the browser is closed. LocalStorage data is stored in the browser's local storage, which is a persistent store that can be accessed across sessions and tabs.

IndexedDB - IndexedDB is a more advanced mechanism for storing data on the client-side that allows for more structured storage and more powerful queries. It is designed for applications that need to store large amounts of structured data, and provides a more robust set of features than sessionStorage or localStorage.
